## hash_without_default

Выявляет использование хэш-блоков (`map`, `geo`) без директивы `default`.

- Важность: Средняя
- Директивы: `map`, `geo`

### Почему это важно

Если в блоке `map` или `geo` отсутствует значение `default`, неожиданные ключи могут приводить к пустым переменным или нежелательным значениям по умолчанию, что потенциально позволяет обходить логику безопасности.

### Пример (сработает)

```nginx
geo $country_code {
  127.0.0.1  ZZ;
}
```

### Пример (безопасно)

```nginx
geo $country_code {
  default    XX;
  127.0.0.1  ZZ;
}
```
